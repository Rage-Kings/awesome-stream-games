## Autoloads

Три глобальных синглтона, объявленных в `project.godot`:

| Синглтон       | Файл                            | Роль                                              |
|----------------|----------------------------------|----------------------------------------------------|
| GameManager    | `src/autoload/game_manager.gd`   | Жизненный цикл мини-игр (LOBBY → PLAYING → RESULTS), глобальный счёт, лидерборд |
| ChatManager    | `src/autoload/chat_manager.gd`   | Подключение к Twitch/Kick, парсинг `!команд`, эмит сигналов |
| PlayerManager  | `src/autoload/player_manager.gd` | Реестр активных игроков, обработка `!join`/`!leave` |

## Поток данных

1. **Чат → Клиент**: `TwitchClient` / `KickClient` получают сообщения через WebSocket
2. **Клиент → ChatManager**: клиенты эмитят сигнал, ChatManager парсит команды (`!команда аргументы`)
3. **ChatManager → PlayerManager**: команды `!join` / `!leave` обрабатываются автоматически
4. **ChatManager → BaseMinigame**: текущая мини-игра получает команды через `_on_chat_command()`
5. **BaseMinigame → GameManager**: по завершении мини-игра отправляет результаты через сигнал `game_finished`
6. **GameManager → UI**: обновляет лидерборд и статус

## Состояния GameManager

```
LOBBY ──► PLAYING ──► RESULTS ──► LOBBY
           │                       ▲
           │   (таймер между       │
           │    играми)            │
           └───────────────────────┘
```

- **LOBBY** — ожидание, игроки могут входить/выходить
- **PLAYING** — мини-игра запущена, принимаются команды
- **RESULTS** — показ результатов, пауза перед следующей игрой
